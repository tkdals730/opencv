| 용어 | 뜻 | 한 줄 설명 |
|------|-----|-----------|
| **색공간 (Color Space)** | 색을 숫자로 표현하는 방법 | BGR은 파랑·초록·빨강 세 숫자, HSV는 색상·채도·명도 세 숫자 |
| **BGR** | Blue·Green·Red | OpenCV가 사용하는 색 순서 (일반적인 RGB와 순서가 반대) |
| **HSV** | Hue·Saturation·Value | 색상(무슨 색) · 채도(얼마나 선명) · 명도(얼마나 밝음) |
| **Hue (색상)** | 색의 종류 | OpenCV에서 0~179 범위. 0=빨강, 60=초록, 120=파랑 |
| **이진화 (Thresholding)** | 기준값으로 흑백 나누기 | 픽셀이 기준보다 밝으면 흰색(255), 어두우면 검정(0) |
| **임계값 (Threshold)** | 흑백을 나누는 기준값 | "이 밝기 이상이면 흰색" 하는 그 경계 숫자 |
| **오츠 (Otsu)** | 자동으로 최적 임계값을 찾는 알고리즘 | 히스토그램을 분석해서 가장 잘 나뉘는 값을 자동 계산 |
| **적응형 이진화 (Adaptive)** | 영역마다 다른 임계값 적용 | 그림자가 있는 이미지에서 한 개 임계값으로는 부족할 때 사용 |
| **ROI (Region of Interest)** | 관심 영역 | 이미지에서 처리하고 싶은 특정 부분만 잘라낸 것 |
| **마스크 (Mask)** | 흑백 이미지로 된 필터 | 흰색 부분만 통과시키고, 검정 부분은 차단하는 역할 |
| **트랙바 (Trackbar)** | 슬라이더 UI | 파라미터 값을 마우스로 실시간 조절할 수 있는 바 |
| **히스토그램 (Histogram)** | 밝기별 픽셀 수 그래프 | 이미지에서 어두운 픽셀이 많은지, 밝은 픽셀이 많은지 한눈에 파악 |
| **크로마키 (Chroma Key)** | 특정 색 배경을 다른 배경으로 교체 | 초록 배경 앞에서 찍고 → 초록색 제거 → 다른 배경 합성 |
